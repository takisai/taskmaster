Task Masterについて

1 概要

Task Masterは、決められた時刻に音を出すことができるWEBツールです。

1.1 諸情報
  • 最新版: Version 0.1 (2019-xx-xx)
  • 製作者: takisai
  • 動作環境: Chrome・Firefox
2 使い方

Task Masterは、基本的に上から2行目の入力欄にコマンド+Enterを入れて動かします。

2.1 タスクの入力

タスクは、決められた時刻になると音を鳴らすなどの動作を行います。タスクは、次の項目を入力することで設定できます。

  1. 時間設定: タイマー式/アラーム式
  2. 通知設定: 音/重要度
  3. タスク名: タスクにつける名前

時間設定は2.1.1で、通知設定は2.1.2で、タスク名は2.1.3で紹介します。

2.1.1 時間設定: タイマー式/アラーム式

タスクが動作する時刻は、動作する時刻までの時間を指定する方法と、時刻を直接指定する方法があります。

タイマーのように「これだけ時間が経ったら音を鳴らす」という設定は、数字を並べて表します。例えば、3分なら3と入力します。また、4分33秒のように秒単位で指定したいときは、4.33と<分>.<秒>の形式で指定します。ただし、<分>は2桁までの数値で、3桁以上の場合は「右側2桁が分、残りが時間」を表します。例えば、1時間5分なら105のようになります。日単位の指定は、時分秒の指定の前に<日>,と書きます。例えば、10,537.43は10日と5時間37分43秒を表しています。

また、タイマー式の指定は<日>d<時間>h<分>m<秒>sの形式でも入力することができます。この指定では1.5hのように、少数での指定もすることができます。

アラームのように「ある時刻になったら」という設定は、時分秒の区切りには:(コロン)、年月日の区切りには-、年月日と時分秒の間には,(コンマ)を入れて表します。例えば、21時38分は21:38、7時38分46秒なら7:38:46、3月7日の22時なら3-7,22:0のように書きます。これらの例のように、年月日や月日、秒は必要なければ省略することができます。

アラームで設定される時間は、書かれたものと等しい一番始めの時刻になります。例えば、朝8時に13:00を指定すると、5時間後にタスクは動作します。一方で、15時に13:00と指定すると、次の日の13時にタスクが動作します。

2.1.2 通知設定: 音/重要度

Task Masterは10種類の音を出すことができます。それぞれ0から9までの数字が割り当てられています。タスクが動作するときの音の設定には、その数字を指定すればOKです。音の確認はsound 0のようにsoundコマンドを使います。

重要度の指定をすると、動作する時刻を過ぎたときのタスクの表示が変わります。

  • 重要度: 低のとき
    .(ピリオド)で表します。重要度: 低のタスクだけは、時間を過ぎると勝手に消えます。
  • 重要度: 中のとき
    !で表します。動作する時刻を過ぎると、そのタスクが黄色で強調されます。
  • 重要度: 高のとき
    !!で表します。動作する時刻を過ぎるとそのタスクは赤色で強調されて、Task Masterの画面全体が赤っぽくなります。
  • 重要度: 超のとき(重要度: 高よりも重要なタスクの設定です)
    !!!で表します。重要度: 高の動作に加えて、ブラウザのアラートが出ます。
    アラートが出ている間は他の処理が止まるため、注意が必要です。

音と重要度の設定は、2.のようにつなげて表現します。例えば、音: 1番、重要度: 中と指定するときは、1!のように指定します。また、これらの指定は必要に応じて省略できます。その場合、省略した設定には既定値が適用されます。

2.1.3 タスク名

タスク名は、;(セミコロン)以外の文字が使えます。省略すると、2.1.1で指定した文字列がタスク名になります。

2.1.4 ここまでのまとめ

2.1.1・2.1.2・2.1.3を/でつないだものが、タスクの指定文字列です。例えば、12時に5番の音を出す重要度低のタスク「昼休み」を設定するときは、12:00/5./昼休みと入力します。また、タスク名は省略できて、その場合は12:00/5.のように/も省略します。また、タスク名に加えて音や重要度の設定も省略するときは、12:00のように最初の/も省きます。

2.1.5 繰り返しタスクを動かす

ここまでのタスクは、一度動作するとそれで終わりというものでした。しかし、繰り返し動くタスクを設定することもできます。例えば、朝6時30分とそれから5分おきに3番の音を鳴らす「GOOD MORNING!」というタスクは6:30+5+/3/GOOD MORNING!のように表します。

繰り返し動くタスクは、10+2+3のように2.1.1で作る文字列を複数+でつなげるだけです。例えば、10+2+3という文字列は10分のタイマーを設定します。10分経って音が鳴るときに、新たに2+3というタスクが設定されます。同様に2分が経過すると、音が鳴ると同時に3というタスクが設定されます。

また、+でつなげる文字列を+で終わらせると、最後の指定がずっと繰り返されるようになります。例えば5+というタスクなら、5分おきに繰り返しタスクが鳴るようになります。

ただし、これで新たに追加されるタスクの音や重要度は、最初に指定したものと同じになります。最初の例(6:30+5+/3/GOOD MORNING!)なら、5分おきに3番の音が鳴る「GOOD MORNING!」というタスクが繰り返し設定されるので、途中で音は変えられません。

2.1.6 指定した時刻に別のコマンドを実行する

タスクで指定した時刻に、別のコマンドを実行することができます。ただし、実行できるコマンドは;や/が入っていないこと、最後の文字が.や!でないことが必要です。例えば、5分後にhogeというコマンドを実行するタスクは5/*hogeのように今までの<音番号>の部分を*<実行したいコマンド>に置き換えます。今までのタスク指定と同様に、重要度やタスク名を設定することもできます。

上の条件に合わないコマンドを実行するためには、2.7で説明するマクロを利用する必要があります。

2.2 タスクの入力(別の方法)

2.1ではタスクの入力方法を紹介しましたが、menu>タスク設定からでも設定することができます。

2.3 タスクの削除

タスクの削除は、2つ方法があります。1つ目は、タスクリストの左側にあるボタンを押す方法です。

もうひとつは、removeコマンドを使う方法です。タスクリストの一番左の番号をremoveの後に書くことで、そのコマンドが消去されます。また、番号は複数書くことができます。例えば、remove 3 4 5なら3から5番目のタスクを削除します。一度にすべてのタスクを消去するには、remove *を入力します。

また、時刻が過ぎたタスクを全て削除するには、番号なしのremoveコマンドを使います。

2.4 タスクの既定値の設定

2.1.2で指定を省略したときの既定値は、defaultコマンドで変更できます。default <音番号><重要度>のように指定します。<音番号><重要度>のどちらも、2.1.2と同じ方法で記述します。また、どちらかを省略することもできます。例えば、default !!は既定の音はそのまま、重要度だけ高に変更するコマンドです。

また、現在設定されている既定値はdefaultで確認することができます。

2.5 タスクの表示方法の切り替え

タスクの後ろには、それがいつ動作するのか時間が表示されています。これを切り替えるには、switchコマンドを使います。

switch alarmとすると、時間の表示は(18:30:00)のような形式になります。これは「18時30分に動作する」という意味です。また、switch timerとすると、時間の表示は[01:53:20]のような形式になります。これは「あと1時間53分20秒で動作する」という意味です。switchとすると、時間の表示を今の設定とは別のものに切り替えます。

2.6 音量操作/音を止める

音量操作は、volumeバーでの操作かvolumeコマンドを使います。volume 50のように、音量を示す数値を指定します。0が音を出さない設定、100は音を一番大きくする設定になります。

Task Masterの音を止める方法は2つあります。1つ目はボタンを押す方法です。ただし、このボタンは音が鳴っているタスクにしか表示されません。

もうひとつはstopコマンドを使う方法です。2.3のremoveコマンドと同様に、タスクリスト一番左の番号を使って、そのタスクの音を止めることができます。また、stop *とすることで、全ての音を止めることができます。

2.7 マクロ

マクロを使うと、例えばramenと入力すれば、自動で3//ramenというコマンドに置き換えて実行することができます。

マクロの指定方法は<元文字列>-><置き換え後文字列>です。ただし、<元文字列>に->を含めることはできません。<元文字列>と<置き換え後文字列>はJavaScriptのreplaceメソッドを利用しているため、複雑な指定をすることができます。例えば、dup (.*)$->$1;button $1というマクロを指定しておくと、dup 50という文字列は50;button 50というコマンドに置き換えられます。詳しくは、JavaScriptのreplaceメソッドを参照してください。

マクロによる置き換えは、登録順にすべてのマクロで行われます。例えば、a->bとb->cというマクロをこの順番に登録した場合、aというコマンドはcに置き換えられます。マクロでの置換が行われないようにするには、先頭に#をつけてコマンドを入力します。

マクロの一覧はshow-macroコマンドで見ることができます。マクロの消去は、マクロリストの左にあるボタンを押すか、remove-macroコマンドにより行います。このコマンドは2.3のremoveコマンドと同様に、remove-macro 1 3 4やremove-macro *と入力できます。マクロ一覧を隠すときはhide-macroを実行します。

参考: JavaScript replace - Google 検索

2.8 式の記述

命令文字列の中に${～}$と記述すると、～の部分がJavaScriptの文として評価されます。たとえば、${24*5}$mという入力は、120mという指定と等しくなります。詳しくは、JavaScriptの文法を参照してください。

参考: JavaScript - Google 検索

2.9 ボタンの作成

今まで書いてきたコマンドは、ボタンを押すことで実行することもできます。buttonコマンドの後ろに入力したいコマンドを入れて実行すると、volume操作バーとタスクリストの間にボタンが現れます。このボタンを押すことで、登録された内容が実行されます。例えば、button switchというコマンドで作成したボタンは、押すたびにswitchコマンドを実行します。ただし、コマンドに;を含めることはできません。

ボタンの削除は、remove-buttonコマンドで行います。2.3のremoveコマンドと同様に、左上のボタンを1としたときのボタンの番号や*で削除を行います。

2.10 消去したものを元に戻す

removeやremove-macro、remove-button、ボタンで消したものは、undoコマンドまたはCtrl+Zで元に戻すことができます。

2.11 一度に複数のコマンドを実行する

マクロを除くこれまでのコマンドは、;(セミコロン)でつなぐことで一度にすべて実行することができます。

2.12 タスクの外部保存

Task Masterはブラウザにデータを保存しているので、履歴の消去などでデータは消えてしまいます。また、同じ設定を入力しない限り、別の端末で同じタスクは管理できません。

saveコマンドまたはCtrl+Sを使うと、Task Masterの全ての設定をセーブデータとして保存することができます。セーブデータを読み込むときは、loadコマンドまたはCtrl+Oを実行して出てくる入力欄にその文字列を貼り付けてください。
