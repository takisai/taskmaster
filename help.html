Task Master Help

1 概要

Task Masterは、WEB上で利用できるリマインダーツールです。

1.1 諸情報
1.2 更新履歴
1.3 ヘルプの読み方

ヘルプは各項目でコマンドの構文や使い方を説明しています。

構文では以下の表記を使います。

  • <分>は、この表記全体を分を表す数値に書き換えるという意味です。
  • [foo]barのように、[]で囲まれた部分は省略可能を意味しています。この例なら、「foobarかbar」という意味です。括弧は入れ子になる場合もあります。
  • foo[bar|baz]の|は、どちらか一方を選択できることを表しています。この例は、「fooかfoobarかfoobaz」という意味です。
1.4 バージョンについて

Task Masterのバージョン番号はX.Y.Zの形式としています。

  • Xはメジャーバージョンを表しています。大きな機能の追加や変更がある場合に、数字が大きくなります。
  • Yはマイナーバージョンを表しています。小さな機能の追加や変更がある場合に、数字が大きくなります。
  • Zはパッチバージョンを表しています。バグの修正があるたびに数字が大きくなります。

メジャー/マイナーバージョンが上がったときには、マクロの動作が変わる可能性があります。

2 使い方 | 基本編
2.1 はじめに

Task Masterは、上から1行目にある時刻と3行目のvolumeバーの間にある入力欄にコマンドを入れることで動かします。上下キーを押して、過去に入力したコマンドをもう一度使うこともできます。

2.2 リマインダーの入力
構文
<時間設定>[/[<通知/時間設定>][/<リマインダー名>]]
<時間設定>
2.2.1や2.2.4の文字列
<通知/時間設定>
2.2.2の文字列
<リマインダー名>
2.2.3の文字列
説明

リマインダーは、決められた時刻になると指定された通りの動作を行います。

例
12:00/5./昼休み
正午に5番の音が鳴る重要度低のリマインダー「昼休み」が登録されます。
17:00+5+
17時から5分おきに音がなる重要度高のリマインダーが登録されます。
2.2.1 時間設定 | タイマー式/アラーム式
構文
  1. [<日>,][<時間>]<分(2桁まで)>[.<秒>]
  2. [<日>d][<時間>h][<分>m][<秒>s] 少数での指定可能 ※全て省略は不可
  3. [[<年>-]<月>-<日>,]<時>:<分>[:<秒>]
説明

リマインダーが動作する時刻を設定します。動作する時刻までの時間を指定する2つの方法と、時刻を直接指定する方法があります。

1と2の書き方は、タイマーのように「これだけ時間が経ったら音を鳴らす」という指定の方法です。1の書き方と比較して、2の書き方は少数での指定ができるなど、より柔軟な指定方法です。

3の書き方は、アラームのように「ある時刻になったら」という指定の方法です。アラームで設定される時間は、書かれたものと等しい一番始めの時刻になります。

例
4.33
4分33秒後を表します。
150
1時間50分後を表します。（150分後ではありません）
5,12
5日と12分後を表します。
6.25h150m
6.25時間と150分後(8時間45分後)を表します。
21:00
21時を表します。21時以前に入力されたならその日の、21時以降に入力されたなら翌日の21時を意味します。
3-7,22:0
3月7日の22時を表します。3月7日22時以降にこの指定を行うと、翌年の3月7日22時という意味になります。
2.2.2 通知/表示設定 | 音/表示方法/重要度/タグ
構文
[<音番号>][<表示方法>][<重要度>][#<タグ文字列>]
<音番号>
1から15までの数値または-(ハイフン)
<表示方法>
aまたはt
<重要度gt;
.(ピリオド)、!、!!、!!!のいずれか
<タグ文字列>
/と;・->・空白を含まず、*(アスタリスク)と等しくない
説明

リマインダーが動作するときの音や時間の表示方法・重要度・タグを設定します。

リマインダーには、それぞれ1から15の数字が割り当てられている15種類の音を設定することができます。リマインダーが動作するときの音の設定には、その数字を指定します。音の確認は3.5.1のsoundコマンドを使います。また、音を出したくない場合は-を指定します。

表示方法の指定をすると、リマインダーの後ろの表示をどうするか決めることができます。aと指定すると、後ろの表示は(18:30:00)のような形式になります。これは「18時30分に音が鳴る」という意味です。また、tとすると、後ろの表示は[01:53:20]のような形式になります。これは「あと1時間53分20秒で音が鳴る」という意味です。表示方法の設定は2.7のswitchコマンドで変更することができます。

重要度の指定をすると、動作する時刻を過ぎたときのリマインダーの表示が変わります。

  • 重要度: 低のとき
    .(ピリオド)で表します。重要度: 低のリマインダーだけは、指定された時刻を過ぎてしばらくすると勝手に消えます。
  • 重要度: 中のとき
    !で表します。指定された時刻を過ぎると、そのリマインダーが黄色で強調されます。
  • 重要度: 高のとき
    !!で表します。指定された時刻を過ぎるとそのリマインダーは赤色で強調されて、Task Masterの画面全体が赤っぽくなります。
  • 重要度: 超のとき(重要度: 高よりも重要なリマインダーの設定です)
    !!!で表します。重要度: 高の動作に加えて、ブラウザのアラートが出ます。
    アラートが出ている間は他の処理が止まるため、注意が必要です。

タグを設定すると、リマインダーがタグ無しのものとは別の場所に表示されるようになります。タグの設定を行わない場合は、タグ無しの場所にリマインダーは表示されます。

タグの設定以外は、省略したとき既定値が設定されます。既定値は2.6のdefaultコマンドで設定できます。

例
2t.#test
音は2番、タイマー方式の表示で、重要度は低、testタグの場所にリマインダーが登録されます。
-!!
音はなし、重要度は高、タグ無しの場所にリマインダーが登録されます。表示方法は既定値が参照されます。
2.2.3 リマインダー名

リマインダー名は、;や->以外の文字が使えます。省略すると、2.2.1で指定した文字列がリマインダー名になります。

2.2.4 繰り返しリマインダーを動かす
構文
<繰り返し時間設定>[+]
<繰り返し時間設定>
2.2.1の文字列を複数+でつないだ文字列
説明

あるリマインダーが鳴るタイミングで、もう一度リマインダーを設定することができます。+で2.2.1の文字列をつなぐと、前の方での指定時間が来るたびに次のリマインダーが登録されていきます。最後に+を入れておくと、最後の指定がずっと繰り返されるようになります。

ただし、これで新たに追加されるリマインダーの音や重要度といった設定は、最初に指定したものと同じになります。設定を変えるには、3.1の指定をすることが必要です。

例
10+2+3
10分後に音が鳴ると同時に、2+3というリマインダーが設定されます。それから2分後にも音が鳴り、同様に3が設定されます。
6:30+5+
6時30分とそれから5分おきに動作するリマインダーを設定します。
2.3 menu > リマインダー設定

2.2ではリマインダーの入力方法を紹介しましたが、menu > リマインダー設定からでも設定することができます。

2.4 リマインダーの削除 | removeコマンド
構文
remove <パラメーター>[ <パラメーター> <パラメーター> ...]
<パラメーター>
次のいずれか
  1. <リマインダー番号>[#<タグ文字列>]
  2. <リマインダー番号>-[#<タグ文字列>]
  3. -<リマインダー番号>[#<タグ文字列>]
  4. <リマインダー番号>-<リマインダー番号>[#<タグ文字列>]
  5. *[#<タグ文字列>|#*]
  6. <重要度>[#<タグ文字列>|#*]
※<リマインダー番号>は全てリマインダーリストの一番左の番号
<重要度>・<タグ文字列>
2.2.2のものと同じ
説明

パラメーターで選択したリマインダーを削除します。パラメーターで指定する数字は、リマインダーリスト一番左にある番号です。

また、リマインダーリストの左側にあるremoveボタンを押すことでも、リマインダーを削除することができます。

  1. <リマインダー番号>[#<タグ文字列>]
    リマインダーを1つ指定します。
  2. <リマインダー番号>-[#<タグ文字列>]
    あるリマインダーから後のもの全てを指定します。
  3. -<リマインダー番号>[#<タグ文字列>]
    あるリマインダーから前のもの全てを指定します。
  4. <リマインダー番号>-<リマインダー番号>[#<タグ文字列>]
    左の数字から右の数字までの範囲のリマインダー全てを指定します。
  5. *[#<タグ文字列>|#*]
    指定したタグの付いた全てのリマインダーを指定します。*のようにタグを指定しない場合はタグ無しのもの全てが、*#*の場合はタグの有無に関係なく全てのリマインダーが指定されます。
  6. <重要度>[#<タグ文字列>|#*]
    5のように指定した範囲の全てのリマインダーでなく、指定した範囲の中で「設定時刻が過ぎて取り消し線の引かれた」リマインダーを指定する方法です。取り消し線の引かれたリマインダーのうち、<重要度>以下のリマインダーが全て指定されます。

間違って消してしまったリマインダーは、2.11のundoコマンドで元に戻すことができます。

例
remove 1 8- 3-5#todo
タグ無しの1番と8番以降、todoタグの付いた3から5番のリマインダーが全て削除されます。
remove !!#*
タグの有無に関係なく、取り消し線の引かれた全リマインダーのうち重要度が高より低いもの全てを削除します。
2.5 タグ | move/remove-tagコマンド

タグは、リマインダーごとに1つ設定することができます。タグごとに表示場所が違い、タグ有りのリマインダーは隠すことができるようになります。リマインダーが登録されるたびに、必要に応じて表示場所は作成されます。

2.5.1 タグの変更 | moveコマンド
構文
move[#<タグ文字列>] <パラメーター>[ <パラメーター> <パラメーター> ...]
#<タグ文字列>
2.2.2の#<タグ文字列>で指定した文字列
<パラメーター>
2.4のパラメーターと同じ
説明

パラメーターで選択したリマインダーのタグを、指定した<タグ文字列>に変更します。タグ文字列の指定がないmoveコマンドは、指定したリマインダーをタグ無しに変更します。これらの変更によって、表示場所も変更されます。

例
move#checked -6
6番までのタグ無しリマインダーをcheckedタグに変更します。
move *#*
タグの有無に関係なく、全てのリマインダーをタグ無しに変更します。
2.5.2 タグの削除 | remove-tagコマンド
構文
remove-tag <パラメーター>[ <パラメーター> <パラメーター> ...]
<パラメーター>
2.4のパラメーターからタグ文字列を除いたものか、2.2.2の#<タグ文字列>。ただし、パラメーターの数字はタグ番号
説明

パラメーターで選択したタグを削除します。タグ番号は、そのタグを上から数えた数字に対応します。また、タグ名の先頭に#を付けることでもタグを指定できます。ただし、リマインダーが1つでも登録されているタグは消すことができません。また、タグの横のremoveボタンを押すことでも同様の動作になります。

間違って消してしまったタグは、2.11のundoコマンドで元に戻すことができます。

例
remove-tag 1 #checked
一番上のタグとcheckedタグを削除します。
remove-tag *
全てのタグを削除します。
2.6 リマインダーの既定値の設定 | defaultコマンド
構文
default [<音番号>][<表示方法>|s][<重要度>]
<音番号>・<表示方法>・<重要度>
2.2.2のものと同じ
説明

2.2.2の通知/表示設定で指定を省略したときの既定値を変更します。指定はどれも2.2.2と同じ方法で記述します。省略した部分の既定値はそのままになります。

表示方法にsを指定すると、現在の表示設定がアラーム式ならタイマー式に、タイマー式ならアラーム式に切り替えることができます。

例
default !
音や表示方法の設定はそのまま、既定の重要度を中に変更します。
default s
音や重要度の設定はそのまま、表示方法を切り替えます。
2.7 リマインダーの表示方法の切り替え | switchコマンド
構文
switch[-alarm|-timer][ <パラメーター> <パラメーター> ...]
<パラメーター>
2.4のパラメーターと同じ
説明

リマインダーの後ろの表示を切り替えます。この表示は、そのリマインダーがいつ鳴るのか/あとどれだけで鳴るのかを示しています。例えば、(09:00:00)は「9時にリマインダーが鳴る」こと、[02:13:04]は「あと2時間13分4秒でリマインダーが鳴る」ことを表しています。パラメーターを省略した場合は、*#*と同じ動作になります。

  • switch-alarmは選択したリマインダーを(09:00:00)のようなアラーム式に変更します。
  • switch-timerは選択したリマインダーを[02:13:04]のようなタイマー式に変更します。
  • switchは選択したリマインダーをタイマー式ならアラーム式に、アラーム式ならタイマー式に切り替えます。

また、リマインダーの後ろの表示を押すことでも表示方式を切り替えることができます。

例
switch-timer 1 1#checked 1#todo
タグ無し1番、checkedタグ1番、todoタグ1番のリマインダーの表示をタイマー式に切り替えます。
switch
全てのリマインダーの表示を現在とは別の表示に切り替えます。
2.8 音を止める | stopコマンド
構文
stop[ <パラメーター> <パラメーター> ...]
<パラメーター>
2.4のパラメーターと同じ
説明

パラメーターで指定したリマインダーの音を止めます。パラメーターを省略した場合は、3.5.1のsoundコマンドで鳴っている音を止めます。また、stopボタンを押すことでも音を止めることができます。

例
stop 1-3
1番から3番のタグ無しリマインダーの音を止めます。
stop *#*
タグの有無に関係なく、全ての音を止めます。
2.9 音量を操作する | volume・muteコマンド

volumeコマンドで音量の大小を、muteコマンドでミュートの有無を変更できます。

2.9.1 音量を操作する | volumeコマンド
構文
volume <数値>
<数値>
0から100の数値
説明

Task Masterの音量を数値で操作します。0が無音で、数値が大きくなるほど音が大きくなります。また、volumeバーで操作することでも音量を操作することができます。

例
volume 30
音量を30に設定します。
2.9.2 音量を操作する | muteコマンド
構文
mute[-on|-off]
説明

Task Masterをミュートするか指定します。mute-onコマンドでミュートになり、mute-offコマンドでミュートしなくなります。muteコマンドは、ミュートの設定を現在のものとは逆にします。

また、volumeバーの左のvolumeの文字を押すことでもミュートのオンオフが操作できます。ミュートしている間は、volumeバーでの音量操作はできません。

例
mute
ミュートになっている場合はミュートを外し、そうでない場合はミュートします。
2.10 ボタン | button/remove-buttonコマンド

ボタンを作成すると、あらかじめ決めておいたコマンドをボタンを押すことで入力・実行することができます。作成したボタンは、入力欄とvolume操作バーの間に現れます。

2.10.1 ボタンの作成 | buttonコマンド
構文
button <コマンド文字列>
<コマンド文字列>
;や->が含まれない文字列
説明

押すことであらかじめ決めておいたコマンドの入力と実行ができるボタンを作成します。

作成したボタンを押すことで、入力した内容が実行されます。例えば、button switchというコマンドで作成したボタンは、押すたびにswitchコマンドを実行します。

例
button switch
押すとswitchが実行されるボタンを作成します。
volume 0
押すと音量が0になるボタンを作成します。
2.10.2 ボタンの削除 | remove-buttonコマンド
構文
remove-button <パラメーター>[ <パラメーター> <パラメーター> ...]
<パラメーター>
2.4のパラメーターと同じ。ただしタグ文字列は無く、数字はボタン番号
説明

パラメーターで選択したボタンを削除します。ボタン番号は、そのボタンを左上から数えた数字に対応します。

間違って消してしまったボタンは、2.11のundoコマンドで元に戻すことができます。

例
remove-button 2
左上から数えて2番目のボタンを削除します。
2.11 消去したものを元に戻す | undoコマンド
構文
undo
説明

removeボタンやremove、remove-tag、remove-button、remove-macroコマンドで消したものを元に戻します。また、Ctrl+Zもこのコマンドと同じように動きます。ただし、ページを読み込み直してしまうと元に戻すことはできなくなります。

2.12 一度に複数のコマンドを実行する
説明

3.2のマクロを除くコマンドは、;(セミコロン)でつなぐことで一度にすべて実行することができます。

例
100;10
1時間後に動くリマインダーと10分後に動くリマインダーを同時に登録します。
switch;default s
全てのリマインダーの時間表示を切り替えて、リマインダー設定時の既定の表示方法を切り替えます。
2.13 セーブデータの外部保存/読み込み | save/load/mergeコマンド

Task Masterはブラウザにデータを保存しているので、cookieなどを消去するとデータが消えてしまいます。また、同じ設定を入力しない限り、別の端末で同じリマインダーは管理できません。

しかし、以下のコマンドを使うとデータの外部保存や読み込みができるようになります。

2.13.1 データの書き出し | saveコマンド
構文
save
説明

Task Masterに保存されているデータをクリップボードにコピーします。ただし、次の情報は含まれません。

  • 削除したリマインダー・タグ・ボタン・マクロ
  • 入力履歴

また、Ctrl+Sでも同様にセーブデータをコピーすることができます。

2.13.2 データの読み込み | loadコマンド
構文
load
説明

2.13.1で取得した文字列でTask Masterの保存データを置き換えます。Ctrl+Oでも同様の動作になります。

2.13.3 データの追加 | mergeコマンド
構文
merge
説明

2.13.1で取得した文字列データのうち、リマインダー・タグ・ボタン・マクロの情報だけを現在の保存データに追加します。また、全く同じ要素は追加されることはありません。Ctrl+Shift+Oでも同様の動作になります。

help

<時間設定>

3.2 マクロ | ->/remove-macro/show-macro/hide-macroコマンド

入力されたコマンドを、前処理をして置き換えることができます。3.3の式の展開より前に行われます。

3.2.1 マクロの登録 | ->コマンド
構文
<置き換え正規表現>-><置き換え後文字列>
<置き換え正規表現>
置き換えたい部分の正規表現
<置き換え後文字列>
置き換えた後に入る文字列
説明

自分で決めたルールに従って、入力された文字列を置き換えることができます。マクロを登録しておけば、2.9や3.1で指定できなかったコマンドでも、マクロで置き換えることで入力することができます。また、マクロは複数登録することができます。複数登録されたマクロでは、登録が古い順に置き換え処理が実行されます。

このコマンドはJavaScriptのString.replaceメソッドを利用しています。詳しくは、JavaScriptのString.replaceメソッドを参照してください。

参考: JavaScript String.replace - Google 検索

例
ラーメン->3//3分経ちました
ラーメンと入力すると、3//3分経ちましたに置き換えるマクロを作成します。ただし、rラーメンと入力すると、このマクロはr3//3分経ちましたと置き換えてしまいます。間違いを防ぐ点では、^ラーメン$->3//3分経ちましたというマクロの方が適切です。このマクロは、ラーメンの前後に文字列が入っていると置き換えることがありません。
^スヌーズ$->sound 1;5+/5!/！！！！
スヌーズと入力すると、sound 1;5+/5!/！！！！に置き換えるマクロを作成します。このマクロが登録されているときに7:00/スヌーズ*!!を入力すると、7時にリマインダーが赤くなり、同時にスヌーズが実行されます。マクロで置き換えられるので7時ちょうどには1番の音が、それ以降5分おきに5番の音がなるようになります。
(\d+)分->$1
<数字>分と入力すると、<数字>に置き換えるマクロを作成します。ただし、このマクロより前に^ラーメン$->3//3分経ちましたというマクロを登録していた場合、意図しない結果となります。
ラーメンという入力をしたとき、ラーメンマクロにより3//3分経ちましたという入力に置き換わります。しかし、ここで3分の部分が置き換わってしまい、最終的には3//3経ちましたになってしまいます。このように、マクロは登録する順番も意識する必要があります。
3.2.2 マクロの削除 | remove-macroコマンド
構文
remove-macro <パラメーター>[ <パラメーター> <パラメーター> ...]
<パラメーター>
2.4のパラメーターと同じ。ただしタグ文字列は無く、数字はマクロ番号
説明

パラメーターで選択したマクロを削除します。マクロ番号は、マクロリストの左の数字に対応します。また、マクロリストの左側にあるremoveボタンを押すことでも削除することができます。

間違って消してしまったボタンは、2.11のundoコマンドで元に戻すことができます。

3.2.3 マクロリストの表示/非表示 | show-macro/hide-macroコマンド
構文
show-macro、hide-macro
説明

マクロリストの表示/非表示を切り替えます。

3.3 式の展開 | ${～$}
構文
${<式>$}
<式>
JavaScriptの式
説明

入力をJavaScriptの式とみなして計算したものに置き換えます。計算できない場合は、その部分を無視します。なお、この置き換えは3.2のマクロより後に行われます。

このコマンドはJavaScriptのevalと同様の機能の関数を利用しています。詳しくは、JavaScriptのeval関数を参照してください。

参考: JavaScript eval - Google 検索

例
${24+12$}//test
36//testという指定に置き換えられます。
^foo (\d+)->${$1*5$}m
マクロと組み合わせた例です。これはfoo <数字>と入力すると、マクロにより${<数字>*5$}mに変換された後、式が展開されます。結果として、入力した数値×5分のリマインダーが設定されます。
3.4 前処理を行わない | #～
構文
#<コマンド文字列>
説明

3.2のマクロや3.3の式の展開を行わないようTask Masterに知らせます。

3.5 その他のコマンド
3.5.1 音を鳴らす | soundコマンド
構文
sound <音番号>
説明

音番号に対応した音を鳴らします。

3.5.2 タグの追加 | tagコマンド
構文
tag <タグ文字列>[ <タグ文字列> <タグ文字列> ...]
<タグ文字列>
2.2.2のものと同じ
説明

新しいタグを追加します。リマインダーの追加やmoveコマンドで新しいタグは自動で登録されるため、通常このコマンドは使う必要はありません。

3.5.3 タグの開閉操作 | open-tag・close-tag・toggle-tagコマンド
構文
<コマンド> <パラメーター>[ <パラメーター> <パラメーター> ...]
<コマンド>
open-tag・close-tag・toggle-tagのいずれか
<パラメーター>
2.4のパラメーターからタグ文字列を除いたものか、2.2.2の#<タグ文字列>。ただし、パラメーターの数字はタグ番号
説明

パラメーターで選択したタグの開閉状態を変更します。open-tagコマンドは開いた状態に、close-tagコマンドは閉じた状態に、toggle-tagコマンドは開閉状態を切り替えます。

3.5.4 削除履歴をリセットする | empty-trashコマンド
構文
empty-trash
説明

2.11のundoコマンドの履歴を消去します。このコマンドを使った後は、それより前に消去したものは復元できなくなります。

3.5.5 menuの表示/非表示 | show-menu/hide-menuコマンド
構文
show-menu、hide-menu
説明

menuの表示/非表示を設定します。

3.5.6 入力履歴をリセットする | empty-historyコマンド
構文
empty-history
説明

入力欄で上下キーを押したときに表示される履歴を消去します。
